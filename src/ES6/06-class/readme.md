# 类：构造函数的语法糖
## 传统构造函数的问题
1. 属性和原型定义方法分离，降低了可读性
2. 原型成员可以被枚举
3. 默认情况下，构造函数仍可以被当做普通函数使用

## 类的特点
1. 类的声明不会被提升，于let const一样存在暂时性死区
2. 类中所有代码均在严格模式下执行
3. 类所有方法都是不可枚举的
4. 累的所有方法内部都无法当做构造函数使用
5. 类的构造器必须使用new来调用s

# 类的继承
< 如果两个类A和B，如果可以描述为B是A，则A和B形成继承关系
如果B是A 则：
1. B 继承自 A
2. A 派生 B
3. B是A的子类
4. A 是 B的父类
如果A是B的父类，则B会自动拥有A中的所有实例成员

新的关键字
- extends：继承，用于类的定义
- super
 - 直接当做函数调用， 表示父类构造函数
 - 如果对象使用，则表示父类的原型
